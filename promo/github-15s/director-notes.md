# LikelySnap GitHub Promo v2 · White Architecture Cut

## Intent

This version replaces the previous UI-heavy promo. It is a 15 second typography-first GitHub film with a white minimal visual system and a stronger Chinese plain-language product narrative.

The film is about one idea:

> LikelySnap is not a skin on OpenScreen. 不是换皮，是底层重做。

The audience should understand the difference in seconds:

- OpenScreen 1.5.0 was a strong base, but its original path was more suited to short recordings.
- Long recordings exposed the practical pain: 录久了怕炸，打开项目干等，导出时机器压力大。
- LikelySnap rebuilds the foundation so long recordings can still open, edit, zoom, follow cursor movement, and export.

## Visual System

- Canvas: `1920 x 1080`.
- Duration: `15s`.
- Style: white/off-white editorial minimalism, black typography, one dark pink accent.
- No fake editor UI, no generic SaaS cards, no gradients as the main event.
- Motion is the design: typographic cuts, text masks, huge words, snapping foundation blocks.
- The “底层重做” reveal should be the emotional peak.

## Story

### 0.0 - 2.2 · Source

Small, honest origin:

`Based on OpenScreen 1.5.0`

Then:

`不是换皮。是底层重做。`

This is not a dunk. It sets up why the rebuild exists.

### 2.2 - 5.2 · Old Failure Mode

Three old pressure points appear as hard black text, then get crossed out:

- `录久了，怕炸`
- `打开项目，干等`
- `导出视频，吃满机器`

The words should feel heavy and slightly too large for the canvas.

### 5.2 - 8.4 · Foundation Break

The screen clears. Huge word:

`底层重做`

Then:

`从录制、打开，到导出`

This is the first big impact.

### 8.4 - 12.2 · New Foundation

Four plain-language blocks snap into a clean vertical stack:

- `长录制不再直接炸`
- `打开项目还能剪`
- `声音 / 摄像头 / 鼠标对得上`
- `导出不再吃爆内存`

Each block should lock in with a crisp physical stop.

### 12.2 - 15.0 · Payoff

The foundation stack collapses into the product line:

`LikelySnap 1.0`

Then:

`长录制，也能剪。`

Final small CTA:

`github.com/Likely7/LikelySnpa`

## Audio

Use a restrained tech bed plus precise SFX:

- low brand impact at the first origin reveal
- hard ticks for old failure points
- stronger impact on `REBUILT`
- block snaps for the plain-language foundation stack
- final logo stamp

GitHub may autoplay muted, so the visual must work silently. The audio is polish, not the carrier of meaning.

## Anti-Slop

- No fake UI.
- No fake metrics.
- No dark SaaS dashboard.
- No purple/blue neon tech background.
- No visual clutter.
- No “features in cards” marketing layout.
- No motion that looks like random slide transitions.
